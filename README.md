# ZeroTermux
这里提供ZeroTermux相关教程，零折腾，有手机有手有ZeroTermux即可


1.ZeroTermux是什么?
ZeroTermux是基于Termux开发的终端模拟器，具备完整的shell访问能力，完整的包管理系统，是Android命令行的首选

2.ZeroTermux能干什么?
简单说:Termux干得了的事情，他能干，Termux很复杂的事情，它很简单。
他在命令行的基础上增加了图形辅助界面，对新手很友好。图形辅助界面里面有很多功能，主要分为几大类:常用， Zero功能， Root功能，美化/UI功能，x11等等。还能便携切换各Shell，还内置了许多镜像源

3.我为什么选中了ZeroTermux
因为它简单易用，并且保留了原有终端。提升命令行体验。

5.正式避坑指南  ──重点

一、安装 & 初始化坑
 
1. 安装 APK 提示“安装失败”“解析包错误”
​
- 安装包下载不完整，重新下载
​
- 系统版本过低，不兼容当前 ZeroTermux
​
- 打开了手机自带的“纯净模式”“风险应用拦截”，关掉再装

​
2. 打开 ZeroTermux 一直卡在初始化 / 黑屏
​
- 后台杀掉重开
​
- 存储空间不足，清内存
​
- 部分手机需要允许“后台运行”“自启动”

​
3. 刚进去就报错、无法执行任何命令
​
- 不要刚打开就狂按回车，等它自己跑完初始化
​
- 初始化没完成就乱操作会直接炸环境
 
 
 
二、权限 & 文件访问坑
 
1. cd /sdcard 找不到、访问不了手机文件
​

- 必须手动给 ZeroTermux 开存储权限
​
- 权限没开，任何访问手机文件的命令都无效

​
2. 复制文件时报错 Permission denied
​
- 路径不对，不要乱抄路径
​
- 不要直接复制到系统目录，先复制到  ~/  主目录

​
3. 解压文件失败、乱码
​
- 文件名不要带中文、空格、特殊符号
​
- 压缩包损坏，重新下载
 
 
 
三、命令 & 软件安装坑
 
1. pkg install 报错、卡半天、失败
​
- 没换源，国内默认源巨慢甚至连不上
​
- 网络问题，切 Wi‑Fi / 流量
​
- 先执行  pkg update ，再装软件
​
2. 命令输对了也说 command not found
​
- 软件根本没装上
​
- 字母打错、大小写错、空格多了
​
- 刚装完没重开终端，偶尔不识别
​
3. 一复制粘贴命令就多出奇怪字符
​
- 不要带网页格式直接粘
​
- 手动手打一遍最稳
 
 
 
四、运行服务 / 后台程序坑
 
1. 开了服务（Redis、SSH 等），一关窗口就停
​
- 普通启动：关窗口 = 进程被杀
​
- 要后台运行必须用对应后台方式
​
2. 端口被占用，启动失败
​
- 上一个程序没正常关掉
​
- 用  kill  关掉进程，或换端口
​
- 不要多个服务抢同一个端口

​
3. 配置文件改完，服务启动不了
​
- 配置文件格式写错、缩进错、中文符号
​
- 复制配置时别把空格、换行弄丢
​
- 用原版干净配置，别乱拼乱改
 
 
 
五、网络 & 连接坑
 
1. 局域网其他设备连不上 ZeroTermux
​
- 手机和电脑必须连同一个 Wi‑Fi
​
- 手机开了热点、流量、VPN 都会影响
​
- 服务要监听  0.0.0.0 ，不能只监听 127.0.0.1
​
2. IP 每次都变，连起来很烦
​
- 路由器重启、换 Wi‑Fi 都会变
​
- 每次连接前先查当前 IP

​
3. SSH 连不上 / 连了没反应
​
- sshd 没启动
​
- 端口不对
​
- 密码没设置
​
- 防火墙/手机安全软件拦截
 
 
 
六、Redis 专属坑（你项目重点）
 
1. redis-server 启动失败
​
- 配置文件路径写错
​
- 配置文件内容错误
​
- 端口 6379 被占用
​
2. redis-cli 连不上
​
- 服务没启动
​
- 绑定 IP 限制了
​
- 设置了密码但没输密码

​
3. Redis 数据重启就没
​
- 没开持久化
​
- 配置里 dir 目录不对，无法写 dump.rdb
 
 
 
七、手机系统专属坑（很重要）
 
1. 杀后台严重，一会儿不用就失效
​
- 电池优化 → 把 ZeroTermux 设为不优化
​
- 锁定后台、加白名单
​
2. 更新系统后 ZeroTermux 用不了
​
- 大版本更新可能破坏环境
​
- 重要配置提前备份

​
3. 多开窗口容易乱
​
- 一个窗口只干一件事
​
- 别在一个窗口疯狂开服务
 
 
 
本指南只为避坑，不做命令教学。
所有内容都是实际使用中真实踩过的坑，适合：
 
- 刚接触 ZeroTermux
​
- 不想折腾环境
​
- 只想稳定跑服务（如 Redis）
​
- 不想到处搜答案的人
